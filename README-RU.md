# simple-name-picker

Английская версия: [README.md](./README.md).

Небольшое веб-приложение на **Vite + React + TypeScript** для **случайного выбора имени** из каталога, **оценки** имён (нравится / не нравится) и **банов**, из-за которых имя пропускается, пока вы его не вернёте. Состояние **сохраняется в браузере** (`localStorage`).

## Возможности

- **Выбор** подставляет случайное имя среди всех, кто не в бане.
- **Нравится** / **Не нравится** меняют сохранённую оценку; затем подставляется **другое случайное имя** (не то, которое вы только что оценили, если есть ещё имена).
- **Бан** убирает текущее имя из пула и сразу выбирает снова; **Вернуть** в списке отброшенных возвращает имя.
- Боковые панели **Отброшенные (в бане)** и **Лучшие по оценке** показывают баны и имена с положительной оценкой (верхний список ограничен по числу строк на экране).
- **Сброс** восстанавливает каталог имён по умолчанию и очищает оценки и баны.

Строка под заголовком показывает число имён в каталоге и сколько из них **в бане среди этого каталога** — тот же набор, что в списке «Отброшенные». Устаревшие записи банов для имён, которых уже нет в списке, там не учитываются.

## Запуск

```bash
npm install
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173).

## Стек

- React 19
- TypeScript (strict)
- Vite 8 (`@vitejs/plugin-react`)
- ESLint (flat config)
- Обычный CSS в `src/` (без UI-фреймворка)

## Сохранение данных

- **Ключ хранилища:** `simple-name-picker:names` (см. `src/namesStorage.ts`).
- **Формат:** `names` (массив строк), `ratings` (объект «имя → число»), `banned` (массив строк). Старые данные, где сохранялся только массив строк, по-прежнему загружаются как список имён с пустыми оценками и банами.
